## wine-cellar

**Intent:** Niko's wine collection — browse, edit, bulk delivery requests, tasting log, Excel pivot analysis. Three interfaces, one Supabase source of truth.

**Runs on:**
- GitHub Pages — Wine Cellar desktop PWA (Vue 3 + ag-grid) and Home Cellar mobile PWA
- GitHub Actions — builds and deploys both apps on every git push
- Excel — Power Query pulls from Supabase (read-only, pivot tables)
- No always-on server needed

**URLs:**
- Desktop: `astridwoloszczuk-dev.github.io/wine-cellar/`
- Mobile: `astridwoloszczuk-dev.github.io/wine-cellar/home/`

**Connects to:**
- Supabase `family-apps` — `wines`, `tastings`, `wine_history` (audit), `settings`

**Credentials needed:**
- Supabase anon key — embedded in frontend (public, read/write via RLS)
- Supabase service role key — only needed for `enrich_wines.py` (one-time enrichment), never committed

**Supabase tables:** `wines`, `tastings`, `wine_history`, `settings` (key: `cellar_config` holds merchant contacts + delivery address)

**Note:** `cellar_config` (merchant emails, delivery address) lives in Supabase `settings` table — NOT in the codebase.
